用户: 小王的老师让他去在Google scholar上搜索一篇资料
但是他经过尝试，无论用手机还是电脑都无法打开这个网站。
小强告诉小王，他有一个"盗版网站"跟那个 Google scholar 的功能是一样的，也能搜索到资料。小王试了一下，发现确实可以用。
不过小王有些疑惑：
① 当他打开这个替代网站时，我的电脑、网络传输传输过程以及远程的服务器等相关的设备上都发生了什么？
②而最始的打不开Google Scholar，问题出现在哪些步骤？

当你打开一个替代网站来访问原本无法打开的 Google Scholar 时，你的电脑、网络传输过程以及远程服务器等相关设备上发生的事情包括：
1. DNS 查询：你的电脑会询问 DNS 服务器，将替代网站的域名解析为 IP 地址。
2. 建立连接：通过 TCP 协议与替代网站的服务器建立连接。
3. 发送请求：你的浏览器发送 HTTP 或 HTTPS 请求到替代网站的服务器。
4. 服务器响应：替代网站的服务器处理请求，并返回相应的数据。
5. 数据传输：数据通过互联网传输回你的电脑，这可能涉及到多个路由器和服务器的中转。
6. 渲染页面：浏览器接收到数据后，渲染页面，供你查看和交互。

而最初无法打开 Google Scholar 的问题，可能出在以下几个步骤：
1. DNS 污染：某些地区的 DNS 服务器可能被篡改，导致对 Google Scholar 的域名解析被故意指向错误的 IP 地址。
2. IP 封锁：Google Scholar 的服务器 IP 地址可能被某些地区的网络防火墙（如中国的长城防火墙）封锁。
3. 内容审查：某些内容可能因为不符合当地法律法规或政策而被屏蔽。
